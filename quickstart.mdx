## Prerequisites

Before you begin, ensure you have the following:

<Callout kind="info">

- A modern web browser like Chrome, Firefox, or Safari
- An email address for account verification
- Optional: A GitHub account for repository integration

</Callout>

## Create Your Account

Follow these steps to sign up and access Annelise Lawson O'BAILEY.

<Steps>
  <Step title="Visit the Dashboard" icon="monitor">
    Navigate to [https://app.anneliselawsonobailey.com](https://app.anneliselawsonobailey.com) and click **Sign Up**.
  </Step>
  <Step title="Enter Your Details" icon="user">
    Provide your email, full name, and a strong password. Accept the terms of service.
  </Step>
  <Step title="Verify Email" icon="mail">
    Check your inbox for a verification link from `no-reply@anneliselawsonobailey.com` and click it to activate your account.
  </Step>
  <Step title="Complete Onboarding" icon="check-circle">
    Log in and complete the quick onboarding quiz to customize your dashboard preferences.
  </Step>
</Steps>

## Configure Your Workspace

Set up your initial workspace based on your needs.

<Tabs>
  <Tab title="Solo Developer" icon="user">
    Select **Solo Workspace** during setup. This provides unlimited docs spaces and basic collaboration.
  </Tab>
  <Tab title="Team Collaboration" icon="users">
    Choose **Team Workspace** and invite up to 5 members via email. Enable shared editing.
  </Tab>
  <Tab title="Enterprise" icon="shield">
    Contact sales for Enterprise setup, which includes SSO, audit logs, and custom branding.
  </Tab>
</Tabs>

<Callout kind="tip">
  You can upgrade your workspace later from the **Settings** menu.
</Callout>

## Create Your First Documentation Space

Now build your first project documentation.

<Steps>
  <Step title="New Space" icon="plus">
    Click **New Space** in the dashboard sidebar. Name it `My Project Docs` and select a template like "API Reference".
  </Step>
  <Step title="Add Pages" icon="file-text">
    Create pages using the **+ Page** button. Start with `introduction.mdx`.
  </Step>
  <Step title="Add Content" icon="edit-3">

    Use the built-in MDX editor to write your first page. Here's a sample:

    <CodeGroup tabs="MDX,Bash">
      ```mdx
      ---
      title: Introduction
      description: Welcome to My Project Docs
      ---

      ## Overview

      This is your first documentation page.
      ```
      ```bash
      # Preview your changes live
      npm run dev
      ```
    </CodeGroup>

  </Step>
  <Step title="Publish" icon="globe">
    Click **Publish** to make your space live at a custom URL like `myproject.anneliselawsonobailey.com`.
  </Step>
</Steps>
